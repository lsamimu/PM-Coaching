# PM Launch Lab — Website

A modern, playful, feminine personal-brand website for **Sammy Lydia** that
positions her as a Product Manager, PM career coach, mentor, and AI-era PM
educator. Built per `directives/pm_launch_lab_Website.md`.

> "Notion + Duolingo + Canva + LinkedIn Creator" — warm, intelligent,
> approachable, and Microsoft-level credible.

## Tech stack
- **Next.js 16** (App Router) + **TypeScript**
- **Tailwind CSS v4** (CSS-based theme in `src/app/globals.css`)
- **Framer Motion** for animations & microinteractions
- `lucide-react` icons (+ inline brand icons in `src/components/ui/BrandIcons.tsx`)

Integrations are scaffolded with clean stubs + `.env` placeholders, ready to
wire up: Supabase, Clerk, Stripe, Resend, Sanity, PostHog, Calendly, ConvertKit/Beehiiv.

## Getting started
```bash
cd website
npm install
cp .env.example .env.local   # optional — site runs fully in stub mode without it
npm run dev                  # http://localhost:3000
```

## Project structure

## Editable content
Everything is editable in one place — no need to touch components:
- **Brand, nav, metrics, links:** `src/lib/site.ts`
- **Services, testimonials, products, blog posts, timeline:** `src/lib/data.ts`

## Job Tracker ↔ crawler connection
The Job Tracker reads `src/data/jobs.json`. Refresh it locally:

```bash
# from the repo root
./scripts/refresh-jobs.sh
# or manually:
.venv/bin/python execution/run_pipeline.py --no-email
```

**Automated:** GitHub Actions (`.github/workflows/pm-jobs.yml`) crawls daily, updates
`jobs.json`, commits to `main`, and emails Excel workbooks. If Vercel is connected to
the repo, each commit redeploys the site with fresh roles.

The export script categorizes roles into PM Internships, Associate PM, Product
Analyst, and Full-Time PM. Search, filter, sort, and save (localStorage) all run
client-side.

## Live PM Roles — Venmo payment

Buyers pay via Venmo, submit their email, and you approve with one click in your inbox.

### Env vars (`.env.local`)

| Variable | Purpose |
|---|---|
| `VENMO_HANDLE` | Your Venmo username (no `@`) |
| `SUPABASE_SERVICE_ROLE_KEY` | Records who has access |
| `RESEND_API_KEY` | Sends approve + access emails |
| `CONTACT_TO` | Your email — receives approve requests |
| `ACCESS_TOKEN_SECRET` | Signs access + approve links |
| `NEXT_PUBLIC_SITE_URL` | Match dev port (e.g. `http://localhost:3001`) |

### Buyer flow

1. `/pm-roles` → **Open Venmo** → pay $19 (include email in note)
2. Submit email → buyer gets “confirming shortly” email
3. You get **Approve access** email → click after checking Venmo
4. Buyer gets access link → full Job Tracker unlocked

## Deploy to Vercel (production)

### 1. Connect the repo
1. Push this project to GitHub
2. [vercel.com](https://vercel.com) → **Add New Project** → import the repo
3. Set **Root Directory** to `website`
4. Deploy

### 2. Environment variables (Vercel → Settings → Environment Variables)

Copy from your local `.env.local`. **Required for PM Roles + forms:**

| Variable | Production value |
|---|---|
| `NEXT_PUBLIC_SITE_URL` | `https://pmlaunchlab.com` |
| `NEXT_PUBLIC_SUPABASE_URL` | Your Supabase project URL |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Supabase publishable/anon key |
| `SUPABASE_SERVICE_ROLE_KEY` | Supabase service_role (secret) |
| `ACCESS_TOKEN_SECRET` | Same random string as local |
| `VENMO_HANDLE` | `Lydia-Sammy` (no `@`) |
| `PM_ROLES_PRICE_CENTS` | `1900` |
| `RESEND_API_KEY` | Resend API key |
| `CONTACT_FROM` | `PM Launch Lab <hello@pmlaunchlab.com>` after domain verified |
| `CONTACT_TO` | Your inbox for approve requests |

**Optional:** `CONVERTKIT_*` (newsletter), `STRIPE_*` (coaching checkout)

### 3. Custom domain
Vercel → **Domains** → add `pmlaunchlab.com` → update DNS at your registrar.

### 4. Resend domain (required for buyer emails)
Resend → **Domains** → verify `pmlaunchlab.com` → use that address in `CONTACT_FROM`.
(`onboarding@resend.dev` only works for testing to your own inbox.)

### 5. GitHub Actions secrets (job crawler email)
Repo → **Settings → Secrets → Actions:** `EMAIL_TO`, `EMAIL_FROM`, `EMAIL_APP_PASSWORD`

### 6. Smoke test after deploy
- `/pm-roles` — Venmo button + submit form
- Approve email → access link → unlocked tracker
- `/contact` — test message arrives
- Job count matches recent crawl
